# L1 — Формальная спецификация MPDT-нейрона

**Дата:** 2026-05-31
**Статус:** Утверждён
**Назначение:** Определить математическую модель, представление, операции и протокол сериализации базового вычислительного элемента системы МАТРИЦА.

---

## 1. Введение

### 1.1. Назначение документа

Настоящий документ определяет **MPDT-нейрон** (McCulloch-Pitts Decision Tree Neuron) — атомарный вычислительный элемент системы МАТРИЦА. MPDT-нейрон является прямым наследником формального нейрона Мак-Каллока–Питтса (1943), но снимает его ключевое ограничение: неспособность реализовать нелинейно разделимые булевы функции. Это достигается заменой линейной пороговой функции на **произвольную булеву функцию**, представленную в виде **обучаемого дерева решений** или эквивалентной **таблицы истинности**.

### 1.2. Место в архитектуре

MPDT-нейрон — это атом, из которого строятся все подсистемы МАТРИЦЫ: рабочая память, долговременная память, планировщик, этический фильтр. Нейрон удовлетворяет всем аксиомам, изложенным в L0:

- **Дискретность:** нейрон оперирует только булевыми сигналами.
- **Локальность:** число входов ограничено константой `K_MAX` (по умолчанию 16, абсолютный максимум 20).
- **Интерпретируемость:** поведение нейрона всегда представимо в виде читаемой логической формулы.
- **Непрерывная эволюция:** нейрон поддерживает операторы мутации и слияния.
- **Безопасность:** неизменяемые аксиомы безопасности кодируются как нейроны с замороженной таблицей истинности.

---

## 2. Формальное определение

### 2.1. Нейрон как кортеж

MPDT-нейрон `N` — это кортеж:

```
N = (id, k, F, S, W, meta)
```

где:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `NeuronId` | Глобально уникальный идентификатор (UUID + поколение) |
| `k` | `int` | Количество входов, `1 ≤ k ≤ K_MAX` |
| `F` | `BooleanFunction` | Логика нейрона: таблица истинности или дерево решений |
| `S` | `NeuronState` | Состояние жизненного цикла |
| `W` | `WeightVector` | Вектор приоритетов входов `w_i ∈ {1,2,3}` |
| `meta` | `Metadata` | Время создания, число мутаций, родительские ID, статистика точности |

### 2.2. Идентификатор нейрона

```
NeuronId {
    uuid: UUID          // неизменен при мутациях
    generation: uint64  // увеличивается при каждой успешной мутации
}
```

### 2.3. Состояния жизненного цикла

| Состояние | Описание | Поведение |
|-----------|----------|-----------|
| `STABLE` | Готов к использованию | Детерминированный булев вывод через таблицу истинности |
| `LEARNING` | Настройка на данных | Может выдавать вероятностный выход через мягкое дерево решений |
| `MUTATING` | Эволюционные испытания | Детерминированный вывод, активная фаза мутаций |
| `FROZEN` | Неизменяемая аксиома безопасности | Детерминированный вывод; мутации запрещены |

## 3. Представление булевой функции

MPDT-нейрон поддерживает два эквивалентных представления `F`:

- **Таблица истинности (Truth Table)** — для быстрого инференса в состояниях `STABLE` и `FROZEN`.
- **Дерево решений (Decision Tree)** — для эволюционного обучения и сжатия в состояниях `LEARNING` и `MUTATING`.

### 3.1. Таблица истинности

Для нейрона с `k` входами таблица истинности — это битовый массив `T` длины `2^k`:

```
T: array[0..2^k-1] of bit
```

Интерпретация: `T[i]` — выход нейрона для входного вектора, представленного как двоичное число `i` (биты от младшего к старшему).

**Вычисление ответа:**

```
function evaluate_LUT(X, T):
    index = 0
    for j in 0..k-1:
        if X[j] == 1:
            index = index | (1 << j)
    return T[index]
```

**Занимаемая память:**
- `k=16`: `2^16 = 65 536` бит = 8 КБ.
- `k=20`: `2^20 = 1 048 576` бит = 128 КБ.

**Свойство:** таблица истинности позволяет реализовать любую булеву функцию от `k` переменных, включая XOR и более сложные.

### 3.2. Дерево решений

Дерево решений — это бинарное дерево, в котором:

- **Внутренний узел `Split`** содержит:
  - `inputIndex ∈ {0..k-1}` — индекс тестируемого бита,
  - `leftChild, rightChild` — поддеревья для случаев, когда бит равен 0 и 1 соответственно.
- **Лист `Leaf`** содержит:
  - `value ∈ {0,1}` — выходное значение.

**Вычисление ответа:**

```
function evaluate_tree(node, X):
    if node is Leaf:
        return node.value
    else if X[node.inputIndex] == 0:
        return evaluate_tree(node.leftChild, X)
    else:
        return evaluate_tree(node.rightChild, X)
```

**Ограничения:**

1. Максимальная глубина `D_max ≤ k`.
2. Запрет повторных проверок одного бита на одном пути от корня к листу.

**Связь с таблицей истинности:** любое дерево решений однозначно задает таблицу истинности путём вычисления для всех `2^k` входов. Обратное преобразование неоднозначно и является задачей минимизации.

## 4. Операции над нейроном

### 4.1. Оценка (Evaluate)

Базовый метод, используемый при инференсе.

**Сигнатура:**
```
output = evaluate(neuron, X)
```
где `X` — входной вектор длины `k`.

**Контракт:**
- Для `STABLE` / `FROZEN`: детерминированный результат через таблицу истинности.
- Для `LEARNING`: результат может быть вероятностным (см. раздел 5.1).

### 4.2. Мутация (Mutate)

Стохастическая операция, изменяющая структуру дерева решений. Используется в генетическом алгоритме.

**Типы мутаций:**

| Оператор | Описание |
|----------|----------|
| `FlipLeaf` | Инвертировать значение в листе |
| `SplitLeaf` | Заменить лист на `Split`, создав два новых листа с тем же значением |
| `PruneTree` | Заменить внутренний узел листом, если оба потомка — листья с одинаковым значением |
| `ChangeInput` | Изменить индекс тестируемого бита во внутреннем узле |
| `SwapChildren` | Поменять местами левое и правое поддеревья |
| `GrowSubtree` | Заменить лист случайным поддеревом глубины 1-3 |
| `Crossover` | Обмен поддеревьями между двумя хромосомами-родителями |
| `CompressBranch` | Заменить поддерево эквивалентной, но более компактной структурой |

**Процедура мутации:**
1. Случайно выбрать один узел с вероятностью, пропорциональной его влиянию на точность.
2. Применить один из доступных операторов с учётом ограничений.
3. Перестроить таблицу истинности из модифицированного дерева.
4. Увеличить `generation` нейрона, сохранив прежний `uuid`.

### 4.3. Слияние (Merge)

Два или более нейрона с одинаковым числом входов могут быть объединены в один.

**Процедура:**
1. Построить объединённую таблицу истинности `T_merged` как результат логической операции над выходами исходных нейронов (например, конъюнкция, дизъюнкция).
2. Создать минимальное дерево решений для `T_merged`.
3. Создать нового нейрона, ссылающегося на родителей.

### 4.4. Сжатие (Simplify)

Операция находит минимальное или близкое к минимальному дерево решений, эквивалентное заданной таблице истинности.

**Алгоритмы:**
- **Точная минимизация:** алгоритм Куайна-МакКласки или Espresso для построения минимальной ДНФ с последующим отображением в дерево.
- **Приближённая минимизация:** генетическое программирование с фитнес-функцией, штрафующей размер дерева.

## 5. Механизмы обучения

### 5.1. Мягкое дерево решений (Soft Decision Tree)

В состоянии `LEARNING` дерево решений интерпретируется как вероятностная модель:

- Каждый лист хранит число `q ∈ [0,1]` — вероятность выдачи `1`.
- Внутренние узлы используют сигмоидную функцию для вероятности перехода в левое/правое поддерево.

Такая модель дифференцируема и может настраиваться стохастическим градиентным спуском. После завершения `LEARNING` мягкое дерево детерминируется (порог 0.5) и компилируется в таблицу истинности. Это не нарушает аксиому дискретности, так как после перехода в `STABLE` все следы непрерывности исчезают.

### 5.2. Эволюционное обучение

Полный цикл обучения нейрона включает:

1. **Инициализация:** случайное небольшое дерево или таблица истинности.
2. **Предварительная настройка (опционально):** перевести в `LEARNING`, обучить мягкое дерево.
3. **Закалка:** перевести в `MUTATING`, детерминировать мягкое дерево.
4. **Эволюционный цикл:** многократно применять мутации, оценивать на отложенной выборке, сохранять лучшие варианты.
5. **Стабилизация:** перевести в `STABLE`, сжать, заморозить при необходимости (`FROZEN`).

---

## 6. Ограничения и инварианты

1. **k ≤ K_MAX.** Нарушение делает нейрон невалидным.
2. **Таблица истинности и дерево решений должны быть консистентны.** При переходе из `LEARNING`/`MUTATING` в `STABLE` таблица пересчитывается из дерева.
3. **Неизменность идентификатора.** `uuid` никогда не меняется; `generation` только увеличивается.
4. **Неприкосновенность FROZEN-нейронов.** Нейроны в состоянии `FROZEN` не могут быть изменены или удалены без внешнего криптографического разрешения.
5. **Детерминизм в STABLE.** Функция `evaluate` для одинакового входа всегда даёт одинаковый выход.
6. **Отсутствие лишних признаков.** Каждый входной бит должен влиять на выход хотя бы для одного входного вектора. В противном случае нейрон подлежит сжатию.

## 7. Формат сериализации

Для сохранения, передачи и распространения нейронов используется формат Apache Avro.

### 7.1. Схема нейрона в состоянии STABLE

```avro
{
  "type": "record",
  "name": "MPDTNeuron",
  "namespace": "io.matrix.neuron",
  "fields": [
    {"name": "id", "type": {
      "type": "record", "name": "NeuronId",
      "fields": [
        {"name": "uuid", "type": "string"},
        {"name": "generation", "type": "long"}
      ]
    }},
    {"name": "k", "type": "int"},
    {"name": "state", "type": {"type": "enum", "name": "NeuronState",
      "symbols": ["STABLE", "LEARNING", "MUTATING", "FROZEN"]
    }},
    {"name": "truthTable", "type": "bytes"},
    {"name": "weights", "type": {"type": "array", "items": "int"}},
    {"name": "metadata", "type": {"type": "record", "name": "Metadata",
      "fields": [
        {"name": "createdAt", "type": "long"},
        {"name": "mutationCount", "type": "long"},
        {"name": "parentIds", "type": {"type": "array", "items": "string"}},
        {"name": "accuracyHistory", "type": {"type": "array", "items": "double"}}
      ]
    }}
  ]
}
```

### 7.2. Примечание

Для состояния `LEARNING` может использоваться отдельная схема, включающая параметры мягкого дерева. При сохранении в снапшот модели нейрон всегда детерминируется в таблицу истинности.

## 8. Правила верификации

Для каждого нейрона должны автоматически проверяться следующие свойства:

- **Полнота таблицы:** длина таблицы равна `2^k`.
- **Консистентность:** дерево, скомпилированное в таблицу, совпадает с хранимой таблицей.
- **Отсутствие лишних признаков:** каждый входной бит влияет на выход.
- **Безопасность:** если нейрон помечен как `FROZEN`, любая попытка мутации или слияния с ним должна отвергаться.
- **Детерминизм:** для `STABLE` и `FROZEN` вызов `evaluate` всегда возвращает одинаковый результат при одинаковых входах.

---

## 9. Заключение

Настоящая спецификация определяет MPDT-нейрон — минимальный, но полный по Тьюрингу в композиции элемент для построения логических когнитивных систем. Его формальная простота и опора на дискретную математику делают систему МАТРИЦА принципиально верифицируемой и энергоэффективной, а встроенные механизмы мутации и сжатия обеспечивают возможность непрерывного обучения и эволюции.
