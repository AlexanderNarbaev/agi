**Дата:** 2026-06-02
**Статус:** Утверждён

---

## 1. Введение

### 1.1. Зачем нужны физические интерфейсы

МАТРИЦА — это не только облачный ИИ. Архитектура на дискретных MPDT-нейронах делает систему пригодной для работы на устройствах с ограниченными ресурсами: от микроконтроллеров с килобайтами памяти до FPGA с прямым отображением логики в железо. Это открывает путь к:

- **Автономным роботам,** которые обучаются на месте, без подключения к облаку.
- **IoT-устройствам,** которые адаптируются к пользователю, не передавая данные вовне.
- **Энергоэффективным вычислениям,** на порядки превосходящим традиционные нейросети.
- **Носимой электронике,** работающей от батарейки месяцами.

### 1.2. Связь с архитектурой

- **L1 (MPDT-нейрон):** таблица истинности естественно отображается на Look-Up Tables (LUT) в FPGA.
- **L3 (FNL):** функциональные доли могут быть скомпилированы в прошивку микроконтроллера.
- **L5 (Сжатие):** сжатая логика минимизирует занимаемую память.
- **L7 (Мультимодальный прокси):** интерфейс с сенсорами и актуаторами.

|---------|-----------|--------|-------------------|---------------------|
| **L0** | Микроконтроллер (ESP32, Arduino) | Килобайты | Микроватты | Умный выключатель, датчик |

## 3. Микроконтроллеры (ESP32, Arduino)

### 3.1. Библиотека `matrix-micro`

Легковесная реализация MPDT-нейрона на C/C++ для микроконтроллеров:

- **TruthTable:** хранится как `const uint8_t[]` или `const uint32_t[]` во flash-памяти.
- **Evaluate:** побитовые операции, время выполнения — десятки наносекунд на ядре ESP32.
- **Обучение:** не выполняется на устройстве; нейрон обучается на сервере, сжимается (L5), затем прошивается.

```c
// Пример для ESP32
#include "mpdt_neuron.h"

// Сжатая таблица истинности для XOR (k=2)
static const uint8_t xor_table[1] = {0b0110};

void setup() {
    MPDTNeuron neuron;
    neuron.k = 2;
    neuron.table = xor_table;
    
    uint8_t input = 0b10; // x1=1, x0=0
    uint8_t output = mpdt_evaluate(&neuron, input); // 1
}
```

### 3.2. Сценарии применения

- **Умный дом:** нейрон, обученный предпочтениям пользователя (освещение, температура), работает на ESP32, не требуя подключения к интернету.
- **Носимые устройства:** фитнес-трекер с нейроном, распознающим паттерны активности.
- **Промышленные датчики:** предсказание отказа оборудования по вибрациям.

### 3.3. Ограничения

- **Обучение:** невозможно на устройстве; нейрон статичен после прошивки. Для адаптации требуется загрузка обновлённой таблицы (OTA).
- **Размер:** для k=16 таблица занимает 8 КБ — приемлемо для большинства микроконтроллеров. Для k=20 (128 КБ) требуется устройство с достаточным объёмом flash.

## 4. Роботы и ROS 2

### 4.1. Шлюз `matrix-ros2-bridge`

Пакет ROS 2, связывающий MPDT-ядро с экосистемой Robot Operating System:

- **Сенсоры:** данные из топиков ROS (LaserScan, Image) преобразуются в бинарные векторы через Сенсорный прокси (L7).
- **Актуаторы:** бинарные выходы нейронов преобразуются в команды управления (Twist, JointTrajectory).
- **Обучение:** генетический алгоритм запускается в симуляции (Gazebo, PyBullet), лучшие нейроны разворачиваются на реальном роботе.

```
rostopic list
/robot/sensor/laser -> matrix_sensor_proxy -> MPDT core
MPDT core -> matrix_effector_proxy -> /robot/cmd_vel
```

### 4.2. Сценарии

- **Мобильный робот:** MPDT-нейроны управляют навигацией, избеганием препятствий, зарядкой.
- **Манипулятор:** обучение захвату объектов через генетический алгоритм (Пилотный проект №4, L13).
- **Рой роботов:** каждый робот имеет локальный инстанс; обмен FNL через Ноосферу (L6) позволяет рою коллективно обучаться.

### 4.3. Преимущества перед традиционными нейросетями

- **Детерминизм:** логика навигации верифицируема.
- **Интерпретируемость:** можно доказать, что робот не нарушит Три запрета (например, не причинит вред человеку).
- **Энергоэффективность:** инференс на FPGA или микроконтроллере продлевает время автономной работы.

## 5. FPGA: прямое отображение логики

### 5.1. Почему FPGA

MPDT-нейрон — это, по сути, таблица истинности. FPGA состоят из Look-Up Tables (LUT), каждая из которых реализует произвольную булеву функцию от нескольких входов (обычно 4–6). Таким образом, один MPDT-нейрон отображается на одну или несколько LUT.

**Преимущества:**
- Задержка инференса: доли наносекунды.
- Энергопотребление: фемтоджоули на операцию (близко к термодинамическому пределу).
- Параллелизм: миллионы нейронов работают одновременно.

### 5.2. Компиляция MPDT в FPGA

**Инструмент:** `matrix-fpga-compiler` — утилита, преобразующая снапшот `.ldn` в прошивку FPGA (Verilog / VHDL).

**Процесс:**
1. Снапшот FNL (L6) загружается в компилятор.
2. Для каждого нейрона генерируется модуль на Verilog, реализующий таблицу истинности как LUT.
3. Топология связей (L3) отображается в соединения между модулями.
4. Результат синтезируется стандартными инструментами FPGA (Vivado, Quartus).

```verilog
// Сгенерированный модуль для XOR (k=2)
module neuron_xor (
    input wire [1:0] inputs,
    output wire out
);
    LUT4 #(
        .INIT(16'b0110100110010110) // Таблица истинности
    ) lut_inst (
        .I0(inputs[0]),
        .I1(inputs[1]),
        .I2(1'b0),
        .I3(1'b0),
        .O(out)
    );
endmodule
```

### 5.3. Прототип на доступной плате

Для демонстрации достаточно недорогой платы (TinyFPGA BX, Lattice iCE40). Она вмещает несколько тысяч LUT, что позволяет реализовать сотни MPDT-нейронов.

### 5.4. Эволюция на FPGA

Обучение (мутации) может выполняться на CPU, а тестирование мутантов — на FPGA для ускорения. Популяция из 100 нейронов оценивается за микросекунды.

## 6. Нейроморфное железо

### 6.1. Существующие платформы

- **Intel Loihi 2:** программируемые нейроны, спайковая коммуникация, обучение на чипе.
- **IBM TrueNorth:** миллион нейронов, сверхнизкое энергопотребление.
- **SpiNNaker:** крупномасштабная симуляция спайковых сетей.

### 6.2. Отображение MPDT на нейроморфные чипы

Хотя MPDT-нейроны не являются спайковыми, их дискретная природа делает отображение возможным:

- Таблица истинности кодируется в синаптические веса и пороги срабатывания.
- Пакетная передача сигналов (L2) заменяется спайковыми пакетами (AER — Address Event Representation).

### 6.3. Исследовательская фаза

Отображение MPDT на нейроморфное железо — задача для Фазы 5 (L8). Требуется исследование:

- Эффективность кодирования булевых функций в спайковые.
- Совместимость с протоколами Loihi (SNN).
- Возможность обучения на чипе (on-chip plasticity).

## 7. Аппаратные ускорители инференса

### 7.1. GPU

Хотя MPDT-нейроны не используют матричные умножения, GPU могут ускорить пакетный инференс за счёт параллелизма. Таблицы истинности хранятся в текстурной памяти, входные векторы обрабатываются тысячами потоков одновременно.

**Применение:** облачные инстансы МАТРИЦЫ, обслуживающие множество пользователей.

### 7.2. ASIC

В перспективе можно разработать специализированный чип, напрямую реализующий MPDT-логику. Это даст максимальную энергоэффективность, но требует значительных инвестиций.

---

## 8. Безопасность физических интерфейсов

### 8.1. Trusted Execution Environment (TEE)

На устройствах с TEE (ARM TrustZone, Intel SGX) FROZEN-нейроны и Этический фильтр (L7) выполняются в защищённой области, недоступной для модификации даже владельцем устройства.

### 8.2. Secure Boot

Прошивка микроконтроллера подписывается, и устройство отказывается загружать неподписанный код. Это предотвращает замену обученного нейрона вредоносной логикой.

### 8.3. Физическая защита

На критически важных объектах FPGA с MPDT-логикой могут быть размещены в корпусах с защитой от вскрытия (tamper-proof).

## 10. Пример: ESP32-нейрон за 5 минут

```bash
# Установить библиотеку
git clone https://github.com/matrix-ai/matrix-micro.git
cd matrix-micro/examples/xor_esp32

# Собрать и прошить
idf.py build
idf.py flash
```

Код примера:

```c
#include "mpdt_neuron.h"
#include "esp_log.h"

static const char *TAG = "mpdt_xor";

// Таблица истинности XOR (k=2)
static const uint8_t xor_table[1] = {0b0110};

void app_main() {
    mpdt_neuron_t neuron;
    mpdt_init(&neuron, 2, xor_table);

    for (int i = 0; i < 4; i++) {
        uint8_t out = mpdt_evaluate(&neuron, i);
        ESP_LOGI(TAG, "Input %d%d -> %d", (i >> 1) & 1, i & 1, out);
    }
}
```

Вывод:

```
I (354) mpdt_xor: Input 00 -> 0
I (354) mpdt_xor: Input 01 -> 1
I (354) mpdt_xor: Input 10 -> 1
I (364) mpdt_xor: Input 11 -> 0
```
