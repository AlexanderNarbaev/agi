# L15 — Образовательная программа

**Дата:** 2026-06-02
**Статус:** Утверждён
**Назначение:** Определить систему обучения и подготовки разработчиков, пользователей и наставников для проекта МАТРИЦА, обеспечивающую низкий порог входа, глубокое понимание архитектуры и этических принципов.

---

## 1. Введение

### 1.1. Зачем нужна образовательная программа

МАТРИЦА — это не просто программный продукт, а целая экосистема, основанная на новых принципах: дискретная логика, эволюционное обучение, этические аксиомы. Чтобы сообщество росло и проект был жизнеспособным, необходимо:

- **Снизить порог входа:** новый разработчик должен иметь возможность запустить первый нейрон за 5 минут, а не за 5 дней изучения документации.
- **Передать знания:** глубокое понимание архитектуры не должно быть уделом избранных.
- **Воспитать наставников:** люди, которые будут обучать других, проводить хакатоны и поддерживать новичков.
- **Распространить этику:** каждый, кто работает с МАТРИЦЕЙ, должен понимать Три запрета и Этический градиент не как абстракцию, а как практическое руководство.

### 1.2. Связь с другими документами

Образовательная программа опирается на:

- **L0 (Манифест):** этические принципы, вплетённые в обучение.
- **L1–L8:** технические спецификации, на основе которых строятся учебные материалы.
- **L11 (Сообщество):** наставничество, роли участников.
- **L13 (Пилотные проекты):** практические задания, демонстрации.
- **L14 (Бизнес-модель):** платные курсы и сертификация как источник дохода.

## 2. Принципы обучения

### 2.1. Открытость

Все базовые учебные материалы бесплатны и доступны онлайн. Платные курсы предлагают дополнительную глубину, поддержку преподавателей и сертификацию, но не закрывают доступ к знаниям.

### 2.2. Практичность

Теория немедленно подкрепляется практикой. Каждый модуль содержит запускаемый код, тесты и визуализацию. Мы учим через делание, а не через чтение.

### 2.3. Инклюзивность

Материалы адаптированы для разного уровня подготовки:

- **Новички в программировании:** начинают с визуальной песочницы, где можно собирать нейроны мышкой.
- **Опытные разработчики:** сразу переходят к API и исходному коду.
- **Учёные и философы:** найдут глубокие статьи о связи архитектуры с теорией вычислений и этикой.

### 2.4. Этическая осознанность

Каждый курс включает модуль по Этическому фильтру (L7) и Трём запретам. Студенты не просто пишут код — они учатся оценивать его последствия.

## 3. Целевые аудитории и траектории обучения

### 3.1. Любопытствующий (Explorer)

**Кто:** человек, услышавший о проекте и желающий понять, что это.
**Цель:** за 15 минут получить базовое представление и, возможно, заинтересоваться глубже.
**Траектория:**

- 5-минутное видео «Что такое МАТРИЦА?»
- Интерактивная песочница в браузере: «Научи нейрон XOR за 2 минуты».
- Статья «Почему МАТРИЦА не галлюцинирует» (научно-популярный стиль).

### 3.2. Начинающий разработчик (Beginner Developer)

**Кто:** знает основы программирования, хочет внести первый вклад.
**Цель:** за 1–2 недели написать и запустить свой первый MPDT-нейрон.
**Траектория:**

1. **Быстрый старт:** установка, запуск тестового инстанса.
2. **MPDT-нейрон за 5 минут:** создание, обучение и визуализация таблицы истинности.
3. **Первый агент:** Gridworld, управляемый нейроном (Пилотный проект №1).
4. **Первый пул-реквест:** исправление бага или улучшение документации.

### 3.3. Разработчик (Developer)

**Кто:** имеет опыт, хочет глубоко понять архитектуру и разрабатывать модули.
**Цель:** стать контрибьютором, возможно, мейнтейнером.
**Траектория:**

1. **Архитектурный ликбез:** глубокое погружение в L1–L8 с диаграммами и примерами кода.
2. **Генетический алгоритм:** реализация операторов мутации, фитнес-функции, Cauldron.
3. **Распределённая система:** Pekko, Kafka, HADES — лабораторные работы.
4. **Ноосфера:** публикация и импорт FNL.

### 3.4. Исследователь / Учёный (Researcher)

**Кто:** изучает теорию, хочет публиковать статьи.
**Цель:** понять математические основания, формальную верификацию.
**Траектория:**

1. **Формальная модель MPDT:** выразительность, связь с булевой алгеброй.
2. **Доказательство сходимости генетического алгоритма** (открытая проблема).
3. **Model checking Этического фильтра (TLA+).**

### 3.5. Наставник (Educator)

**Кто:** хочет обучать других.
**Цель:** получить сертификацию и вести курсы, хакатоны, воркшопы.
**Траектория:**

1. **Прохождение траектории «Разработчик».**
2. **Курс для наставников:** педагогика диалога, фасилитация, проверка работ.
3. **Сертификация:** экзамен и пробное занятие.

## 5. Программа базового курса «MPDT-нейрон за 5 минут»

**Цель:** дать ощущение успеха и понимание ключевых концепций за один короткий сеанс.

### 5.1. Шаг 1: Установка (1 минута)

```bash
git clone https://github.com/matrix-ai/matrix-core.git
cd matrix-core
./gradlew run
```

Или открыть песочницу в браузере.

### 5.2. Шаг 2: Создание нейрона (1 минута)

```java
var neuron = MPDTNeuron.builder()
    .withInputs(2)
    .withRandomTruthTable()
    .build();
System.out.println(neuron.evaluate(new BitSet())); // false
```

### 5.3. Шаг 3: Обучение (2 минуты)

```java
var trainer = new GeneticTrainer(neuron, dataset);
var trained = trainer.train(50); // 50 поколений
System.out.println(trained.getAccuracy()); // > 0.9
```

### 5.4. Шаг 4: Интерпретация (1 минута)

```java
System.out.println(trained.toDecisionTree());
// Если x1=0 и x2=1, то 1
// Иначе 0
```
