### How to become a university partner

1. Review `docs/L22.md` (Partnerships with Universities)
2. Open a GitHub issue with label `partnership`
3. Describe: your institution, research group, proposed collaboration
4. We'll schedule an introductory call within 2 weeks

### Collaboration types
- **Research:** Joint publications, grant applications, student projects
- **Curriculum:** Pilot course using MATRIX in your CS/AI program
- **Internship:** Students contribute to MATRIX as part of their degree
- **Facility:** Access to hardware (FPGA, robots) for MATRIX testing

### How to become a corporate partner

1. Review `docs/L23.md` (Corporate Participants) and `docs/L12_Legal.md`
2. Complete the Ethical Questionnaire (in `docs/L23.md` §4.2)
3. Open a GitHub issue with label `corporate`
4. Ethics Committee review (2 weeks)

### Corporate participation types
- **Sponsor:** Financial support, logo on website, priority support
- **Contributor:** Employees contribute code as part of work time
- **User:** Using MATRIX in production (self-hosted or Managed Matrix)
- **Partner:** Joint product development, co-marketing
